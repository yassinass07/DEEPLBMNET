# Security Policy

## Supported Versions

The table below indicates which versions of DEEPLBMNet are currently supported with security updates.

| Version | Supported |
| ------- | --------- |
| 0.1.x   | ✅ Yes     |
| < 0.1   | ❌ No      |

---

## Reporting a Vulnerability

The security of DEEPLBMNet is important.

If you discover a potential security vulnerability, please **do not open a public GitHub Issue**.

Instead, please report it privately through the official website:

**https://deeplbmnet.com**

Include the following information whenever possible:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested mitigation (if available)
* Screenshots or logs (optional)

---

## Response Process

Every security report will be:

* Acknowledged as soon as possible.
* Investigated carefully.
* Verified and reproduced when applicable.
* Addressed in a future security update if confirmed.

We appreciate responsible disclosure and will work to resolve verified issues in a timely manner.

---

## Scope

This policy applies to:

* Source code
* Documentation
* Released software
* Future open-source components of DEEPLBMNet

---

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing any confirmed security issue.

Responsible disclosure helps protect both users and the research community.

---

## Thank You

Thank you for helping improve the security and reliability of DEEPLBMNet.
